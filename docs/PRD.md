# 마이링크 (My Link) — 기능 정의서 (PRD)

## 1. 개요

마이링크는 흩어져 있는 여러 링크(SNS, 블로그, 포트폴리오 등)를 하나의 프로필 페이지에 모아 공유하는 **링크-인-바이오** 서비스다. 사용자는 로그인 후 자신만의 공개 URL을 받아 링크를 자유롭게 관리하고, 방문자는 그 URL로 들어와 원하는 링크로 이동한다.

| 항목 | 내용 |
|------|------|
| 프로젝트명 | 마이링크 (My Link) |
| 한 줄 소개 | 내 모든 링크를 한 페이지에 모아 공유하는 링크-인-바이오 |
| 대상 사용자 | 개발자, 크리에이터 등 SNS·콘텐츠를 여러 곳에 운영하는 사람 |
| 핵심 가치 | 로그인 후 1분 안에 나만의 링크 페이지를 만들어 공유 |
| 기술 스택 | Next.js (App Router) · Tailwind CSS · Firebase (Auth/Firestore) · Vercel |

---

## 2. 풀려는 문제와 목표

- **문제**: 인스타·유튜브·블로그·깃허브 등 링크가 흩어져 있어 한 번에 공유하기 어렵다. SNS 프로필에는 보통 링크를 하나만 넣을 수 있다.
- **목표**: "내 링크 모음" 한 페이지를 만들고, 그 URL 하나만 공유하면 끝나도록 한다.
- **성공 기준**: 가입~첫 링크 추가까지 1분 이내, 공개 페이지 초기 로딩 1초 이내.

## 5. 기능 상세

### F1. Google 로그인

- **동작**: 로그인 페이지에서 "Google로 로그인" 클릭 → Google 인증 팝업 → 완료 시 관리 페이지로 이동.
- **최초 로그인**: 계정 정보가 없으면 초기 설정 페이지(`/setup`)로 보내 공개 URL(slug)을 정하게 한다.
- **초기값**: 이름·프로필 이미지는 Google 계정 값을 그대로 가져온다.
- **예외**: 인증 취소·실패 시 로그인 페이지에 머무르고 안내 메시지 표시.

### F2. 프로필 설정

- **이름(name)**: 화면에 표시되는 사용자 이름. 수정 가능.
- **공개 URL(slug)**: `/{slug}` 형태로 쓰이는 고유 주소. 영문·숫자만, 중복 불가. 기본값은 Gmail 아이디(@ 앞부분)로 자동 제안.
- **소개글(bio)**: 최대 150자. 수정 가능.
- **프로필 이미지**: Google 계정 이미지 사용 (별도 업로드 없음).
- **예외**: slug가 이미 쓰이는 값이면 저장 막고 경고 표시.

### F3. 링크 CRUD

- **추가**: 제목 + URL 입력 → 저장 → 목록에 즉시 반영.
- **수정**: 목록에서 인라인 편집으로 제목·URL 변경.
- **삭제**: 삭제 버튼 → 확인 다이얼로그 → 확정 시 제거.
- **아이콘**: URL 도메인으로 파비콘 자동 표시 (Google Favicon API).
- **제한**: 1인당 최대 20개.
- **예외**: URL 형식이 올바르지 않으면 저장 막고 경고 표시.

### F4. 공개 프로필 페이지

- **주소**: `/{slug}` — 로그인 없이 접근 가능.
- **구성**: 프로필 이미지 → 이름 → 소개글 → 링크 버튼 목록(파비콘 포함).
- **링크 클릭**: 새 탭에서 해당 URL 열기 (F6 활성 시 클릭 수 +1).
- **예외**: 없는 slug면 404 안내 페이지.

### F5. 반응형 레이아웃

- 모바일 퍼스트.
- 브레이크포인트: 모바일 360~767px / 태블릿 768~1023px / 데스크톱 1024px~.
- 카드 폭: 모바일 100%(좌우 패딩 16px) / 태블릿 80%(최대 480px) / 데스크톱 400px 고정, 항상 중앙 정렬.

### F6. 링크 클릭 수 (선택)

- 링크별 클릭 횟수를 누적 저장, 관리 페이지에서 확인.

### F7. 링크 순서 변경 (선택)

- 드래그로 순서를 바꾸고 그 순서대로 공개 페이지에 노출.

## 6. 디자인 — Neobrutalism

전체 화면을 Neobrutalism 스타일로 통일한다. 굵은 검은 테두리, 블러 없는 하드 그림자, 밝고 대비되는 색이 핵심이다.

| 토큰 | 값 |
|------|------|
| 테두리 | 3px solid black |
| 하드 그림자 — 카드 | 6px 6px 0 black |
| 하드 그림자 — 버튼 | 4px 4px 0 black |
| 카드 배경 | `#FEF08A` (노랑) |
| 페이지 배경 | `#E0E7FF` (연보라) |
| 버튼 배경 | `#FFFFFF` |
| 모서리 | 카드 12px / 버튼 8px |
| 프로필 이미지 | 원형 120px, 3px solid black 테두리 |

- 버튼 호버 시 2px 우하향 이동 + 그림자 제거 → 눌리는 느낌.
- 공개 페이지·관리 화면 모두 같은 토큰을 사용해 일관성 유지.

## 7. 데이터 모델 (Firestore)

```
users/{uid}
├── name: string         // 표시 이름
├── slug: string         // 공개 URL (고유)
├── bio: string          // 소개글 (최대 150자)
├── photoURL: string     // Google 프로필 이미지
├── createdAt: timestamp
│
└── links/{linkId}       // 서브컬렉션
    ├── title: string
    ├── url: string
    ├── order: number     // 정렬 순서 (F7)
    ├── clicks: number    // 클릭 수 (F6)
    └── createdAt: timestamp
```
